## bp
履歴管理関連の API を提供する Blueprint インスタンス。
名前は `"history"` 、URL プレフィックスは [`router:create_api_blueprint`](./02.router.md#create_api_blueprint) によって `"/api/history"` に設定される。

#### シグネチャ
```python[history.py]
bp: Blueprint
```

#### 依存するライブラリ
- **Flask**： Blueprint を使用する。

## filter_options()
履歴データの空フィルターを取得する。

#### シグネチャ
```python[history.py]
@bp.get("/filter-options")
@login_required
@roles_required(USER_ROLES.SYSTEM_ADMIN, USER_ROLES.REPOSITORY_ADMIN)
@validate(response_many=True)
def filter_options() -> list[FilterOption]:
```

#### エンドポイント
**GET** /api/history/filter-options

#### 依存するライブラリ
- **Flask**： Blueprint を使用する。
- **Flask-Login**： ログインユーザーの認証を行う。
- **Flask-Pydantic**： リクエストとレスポンスのバリデーション、シリアライズ・デシリアライズを行う。

#### デコレータ
- `@bp.get`{lang=python}： GET リクエストを処理するエンドポイントを定義する。
- `@login_required`{lang=python}： ログインを要求する。
- `@roles_required`{lang=python}： ログインユーザーに「システム管理者」または「リポジトリ管理者」のロールを要求する。
- `@validate`{lang=python}： モデルクラスを使用してリクエストとレスポンスのバリデーションを行う。

#### 戻り値
| 型                                                                      | 説明                   |
| ----------------------------------------------------------------------- | ---------------------- |
| list\[[FilterOption](../03.entities/07.search-request.md#filteroption)] | 履歴データのフィルター |

#### 処理内容
1. [`services.utils::search_history_filter_options`](../05.services/09.utils.md#search_history_filter_options) を実行して選択肢が空のフィルターを取得する。
2. list\[[FilterOption](../03.entities/07.search-request.md#filteroption)] を戻り値として返す。

## filter_options_operators
履歴データの実行ユーザーのフィルター選択肢を取得する。

#### シグネチャ
```python[history.py]
@bp.get("/<tab>/filter-options/operators")
@login_required
@roles_required(USER_ROLES.SYSTEM_ADMIN, USER_ROLES.REPOSITORY_ADMIN)
@validate(response_by_alias=True, exclude_none=True)
def filter_options_operators(tab: t.Literal["download", "upload"], query: OperatorQuery) -> tuple[SearchResult, int] | tuple[ErrorResponse, int]:
```

#### エンドポイント
**GET** /api/history/<tab>/filter-options/operators

#### 依存するライブラリ
- **Flask**： Blueprint を使用する。
- **Flask-Login**： ログインユーザーの認証を行う。
- **Flask-Pydantic**： リクエストとレスポンスのバリデーション、シリアライズ・デシリアライズを行う。

#### デコレータ
- `@bp.get`{lang=python}： GET リクエストを処理するエンドポイントを定義する。
- `@login_required`{lang=python}： ログインを要求する。
- `@roles_required`{lang=python}： ログインユーザーに「システム管理者」または「リポジトリ管理者」のロールを要求する。
- `@validate`{lang=python}： モデルクラスを使用してリクエストとレスポンスのバリデーションを行う。

#### 引数
| 名前  | 型                                             | 説明             |
| ----- | ---------------------------------------------- | ---------------- |
| tab   | Literal\["download","upload"]                  | 履歴のタイプ     |
| query | [OperatorQuery](./11.schemas.md#operatorquery) | クエリパラメータ |

#### 戻り値
| 型                                                                            | 説明                               |
| ----------------------------------------------------------------------------- | ---------------------------------- |
| tuple\[[SearchResult](../03.entities/07.search-request.md#searchresult), int] | 検索結果とステータスコード         |
| tuple\[[ErrorResponse](./11.schemas.md#errorresponse), int]                   | エラーレスポンスとステータスコード |

#### 処理内容
1. クエリパラメータから [`OperatorQuery`](./11.schemas.md#operatorquery) を取得する。
2. `tab` 、 `key="0"`、 [`OperatorQuery`](./11.schemas.md#operatorquery) をもとに [`services.history::get_filter_items`](../05.services/07.history.md#get_filter_items) で実行ユーザーのフィルター選択肢を取得する。
3. [`SearchResult`](../03.entities/07.search-request.md#searchresult) とステータスコード 200 を戻り値として返す。
4. `key` が無効な場合は [`ErrorResponse`](./11.schemas.md#errorresponse) とステータスコード 400 を戻り値として返す。


## get
閲覧可能な履歴データを取得する。

#### シグネチャ
```python[history.py]
@bp.get("/<string:tab>")
@login_required
@roles_required(USER_ROLES.SYSTEM_ADMIN, USER_ROLES.REPOSITORY_ADMIN)
@validate(response_by_alias=True, exclude_none=True)
def get(tab:Literal["download","upload"], query: HistoryQuery) -> tuple[SearchResult, int]:
```

#### エンドポイント
**GET** /api/history/<tab>

#### 依存するライブラリ
- **Flask**： Blueprint を使用する。
- **Flask-Login**： ログインユーザーの認証を行う。
- **Flask-Pydantic**： リクエストとレスポンスのバリデーション、シリアライズ・デシリアライズを行う。

#### デコレータ
- `@bp.get`{lang=python}： GET リクエストを処理するエンドポイントを定義する。
- `@login_required`{lang=python}： ログインを要求する。
- `@roles_required`{lang=python}： ログインユーザーに「システム管理者」または「リポジトリ管理者」のロールを要求する。
- `@validate`{lang=python}： モデルクラスを使用してリクエストとレスポンスのバリデーションを行う。

#### 引数
| 名前  | 型                                                               | 説明           |
| ----- | ---------------------------------------------------------------- | -------------- |
| tab   | Literal\["download","upload"]                                    | 履歴のタイプ   |
| query | [HistoryQuery](../03.entities/14.history-detail.md#historyquery) | フィルター項目 |

#### 戻り値
| 型                                                                            | 説明                       |
| ----------------------------------------------------------------------------- | -------------------------- |
| tuple\[[SearchResult](../03.entities/07.search-request.md#searchresult), int] | 検索結果とステータスコード |

#### 処理内容
1. クエリパラメータから [`HistoryQuery`](../03.entities/14.history-detail.md#historyquery) を取得する。
2. `tab="download"` ならば [`HistoryQuery`](../03.entities/14.history-detail.md#historyquery) をもとに`services.history::get_download_history_data()`で履歴データを取得する。
3. `tab="download"` でないならば [`HistoryQuery`](../03.entities/14.history-detail.md#historyquery) をもとに`services.history::get_upload_history_data()`で履歴データを取得する。
4. [`SearchResult`](../03.entities/07.search-request.md#searchresult) とステータスコード 200 を戻り値として返す。

## public_status
履歴データの公開/非公開を変更する

#### シグネチャ
```python[history.py]
@bp.put("/<tab>/<history_id>/public-status")
@login_required
@roles_required(USER_ROLES.SYSTEM_ADMIN)
@validate(response_by_alias=True)
def public_status(tab: t.Literal["download", "upload"], history_id: UUID, body: HistoryPublic) -> tuple[HistoryPublic | ErrorResponse, int]:
```

#### エンドポイント
**PUT** /api/history/<tab>/<history_id>/public-status

#### 依存するライブラリ
- **Flask**： Blueprint を使用する。
- **Flask-Login**： ログインユーザーの認証を行う。
- **Flask-Pydantic**： リクエストとレスポンスのバリデーション、シリアライズ・デシリアライズを行う。

#### デコレータ
- `@bp.put`{lang=python}： PUT リクエストを処理するエンドポイントを定義する。
- `@login_required`{lang=python}： ログインを要求する。
- `@roles_required`{lang=python}： ログインユーザーに「システム管理者」のロールを要求する。
- `@validate`{lang=python}： モデルクラスを使用してリクエストとレスポンスのバリデーションを行う。

#### 引数
| 名前       | 型                            | 説明                             |
| ---------- | ----------------------------- | -------------------------------- |
| history_id | uuid.UUID                     | 公開設定を変更する履歴の ID      |
| tab        | Literal\["upload","download"] | 履歴のタイプ                     |
| body       | HistoryPublic                 | 設定する値を持つリクエストボディ |

#### 戻り値
| 型                                                                    | 説明                           |
| --------------------------------------------------------------------- | ------------------------------ |
| tuple\[[HistoryPublic](./11.schemas.md#historypublic), Literal\[200]] | 変更後のpublicの値             |
| tuple\[[ErrorResponse](./11.schemas.md#errorresponse), Literal\[404]] | 指定された履歴が存在しない場合 |


#### 処理内容
1. パスパラメータ `tab` からダウンロードかアップロードか判断する。
2. `tab` を `history_type` として `services.history::update_public_status()` を実行する。
3. 戻り値を戻り値として返す。
4. [`RecordNotFound`](../02.foundation/exc.md#recordnotfound) が発生した場合は [`ErrorResponse`](./11.schemas.md#errorresponse) とステータスコード 404 を戻り値として返す。

## files
ファイルをダウンロードするエンドポイント

#### シグネチャ
```python[history.py]
@bp.get("/files/<file_id>")
@login_required
@roles_required(USER_ROLES.SYSTEM_ADMIN, USER_ROLES.REPOSITORY_ADMIN)
@validate(response_by_alias=True)
def files(file_id: UUID) -> Response | tuple[ErrorResponse, int]:
```

#### エンドポイント
**GET** api/history/files/<file_id>

#### 依存するライブラリ
- **Flask**： Blueprint、send_file を使用する。
- **Flask-Login**： ログインユーザーの認証を行う。
- **Flask-Pydantic**： リクエストとレスポンスのバリデーション、シリアライズ・デシリアライズを行う。
- **pathlib**： ファイルの存在確認を行う。

#### デコレータ
- `@bp.get`{lang=python}： GET リクエストを処理するエンドポイントを定義する。
- `@login_required`{lang=python}： ログインを要求する。
- `@roles_required`{lang=python}： ログインユーザーに「システム管理者」または「リポジトリ管理者」のロールを要求する。
- `@validate`{lang=python}： モデルクラスを使用してリクエストとレスポンスのバリデーションを行う。

#### 引数
| 名前    | 型        | 説明       |
| ------- | --------- | ---------- |
| file_id | uuid.UUID | ファイルid |

#### 戻り値
| 型                                                                    | 説明                               |
| --------------------------------------------------------------------- | ---------------------------------- |
| Response                                                              | ファイルを含めたレスポンス         |
| tuple\[[ErrorResponse](./11.schemas.md#errorresponse), Literal\[404]] | 指定されたファイルが存在しない場合 |

#### 処理内容
1. `file_id` をもとに [`services.history::get_file_path`](../05.services/07.history.md#get_file_path) を呼び出し [`files`](../../03.db/02.history.md#files-テーブル) テーブルから `file_path` を取得する。
2. [`RecordNotFound`](../02.foundation/exc.md#recordnotfound) が発生した場合は [`ErrorResponse`](./11.schemas.md#errorresponse) とステータスコード 404 を戻り値として返す。
3. `file_path` をもとにファイルが存在するか確認する。
4. ファイルが存在しない場合は [`ErrorResponse`](./11.schemas.md#errorresponse) とステータスコード 404 を戻り値として返す。
5. ファイルが存在する場合は `send_file()` でファイルを戻り値として返す。
