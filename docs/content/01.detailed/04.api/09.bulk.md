## bp
グループ管理関連の API を提供する Blueprint インスタンス。
名前は `"bulk"` 、URL プレフィックスは [`router:create_api_blueprint`](./02.router.md#create_api_blueprint) によって `"/api/bulk"` に設定される。

#### シグネチャ
```python[bulk.py]
bp: Blueprint
```

#### 依存するライブラリ
- **Flask**： Blueprint を使用する。

## upload_file
一括操作するファイルをサーバーにアップロードしバリデーションタスクを開始するエンドポイント

#### シグネチャ
```python[bulk.py]
@bp.post("/upload-file")
@login_required
@roles_required(USER_ROLES.SYSTEM_ADMIN, USER_ROLES.REPOSITORY_ADMIN)
@validate_files
@validate(response_by_alias=True)
@require_enabled("enable_bulk_operation")
def upload_file(form: TargetRepositoryForm, files: BulkFileForm) -> tuple[BulkBody | ErrorResponse, int]:
```

#### エンドポイント
**POST** /api/bulk/upload_file

#### 依存するライブラリ
- **Flask**： Blueprint を使用する。
- **Flask-Login**： ログインユーザーの認証を行う。
- **Flask-Pydantic**： リクエストとレスポンスのバリデーション、シリアライズ・デシリアライズを行う。

#### デコレータ
- `@bp.post`{lang=python}： POST リクエストを処理するエンドポイントを定義する。
- `@login_required`{lang=python}： ログインを要求する。
- `@roles_required`{lang=python}： ログインユーザーに「システム管理者」または「リポジトリ管理者」のロールを要求する。
- `@validate`{lang=python}： モデルクラスを使用してリクエストとレスポンスのバリデーションを行う。
- `@validate_files`{lang=python}: `multipart/form-data` からファイルを取得、検証し、データクラスにパッケージ化して関数の引数に注入するデコレータ。
- `@require_enabled("enable_bulk_operation")`{lang=python}： サーバー設定の `enable_bulk_operation` が `True` の場合のみエンドポイントを有効にする。

#### 引数
| 名前  | 型                                                                   | 説明                       |
| ----- | -------------------------------------------------------------------- | -------------------------- |
| form  | [TargetRepositoryForm](../04.api/11.schemas.md#targetrepositoryform) | 選択されたリポジトリ ID    |
| files | [BulkFileForm](../04.api/11.schemas.md#bulkfileform)                 | アップロードされたファイル |

#### 戻り値
| 型                                                                           | 説明                                       |
| ---------------------------------------------------------------------------- | ------------------------------------------ |
| tuple\[[BulkBody](../04.api/11.schemas.md#bulkbody),Literal\[200]]           | タスク ID とファイルの識別子               |
| tuple\[[ErrorResponse](../04.api/11.schemas.md#errorresponse),Literal\[400]] | ファイルに不備がある場合のエラーレスポンス |

#### 処理内容
1. [`TargetRepositoryForm`](../04.api/11.schemas.md#targetrepositoryform) 内の選択されたリポジトリ ID を引数に [`services.repository:get_repository_by_id`](../05.services/08.repository.md#get_repository_by_id) を呼び出しリポジトリの存在を確認する。存在しない場合は [`ErrorResponse`](../04.api/11.schemas.md#errorresponse) とともに 400 を戻り値として返す。
2. ユーザーが対象のリポジトリに対して権限があるか確認する。権限がない場合は [`ErrorResponse`](../04.api/11.schemas.md#errorresponse) とともに 403 を戻り値として返す。
3. [`services.bulk:upload_file`](../05.services/05.bulk.md#upload_file) を呼び出しファイルを一時保存して `temporary_id` を取得する。
4. [`services.bulk:validate_upload_data`](../05.services/05.bulk.md#validate_upload_data) を呼び出しファイルのバリデーションを行う `celery` のタスクを実行する。
5. タスク ID と `temporary_id` を [`BulkBody`](../04.api/11.schemas.md#bulkbody) に入れ戻り値として返す。

## validate_status
バリデーションタスクの実行状況の取得エンドポイント

#### シグネチャ
```python[bulk.py]
@bp.get("/validate/status/<string:task_id>")
@login_required
@roles_required(USER_ROLES.SYSTEM_ADMIN, USER_ROLES.REPOSITORY_ADMIN)
@validate(response_by_alias=True)
@require_enabled("enable_bulk_operation")
def validate_status(task_id: str) -> tuple[BulkBody | ErrorResponse, int]:
```

#### エンドポイント
**GET** /api/bulk/validate/status/<task_id>

#### 依存するライブラリ
- **Flask**： Blueprint を使用する。
- **Flask-Login**： ログインユーザーの認証を行う。
- **Flask-Pydantic**： リクエストとレスポンスのバリデーション、シリアライズ・デシリアライズを行う。

#### デコレータ
- `@bp.get`{lang=python}： GET リクエストを処理するエンドポイントを定義する。
- `@login_required`{lang=python}： ログインを要求する。
- `@roles_required`{lang=python}： ログインユーザーに「システム管理者」または「リポジトリ管理者」のロールを要求する。
- `@validate`{lang=python}： モデルクラスを使用してリクエストとレスポンスのバリデーションを行う。
- `@require_enabled`{lang=python}： サーバー設定の `enable_bulk_operation` が `True` の場合のみエンドポイントを有効にする。

#### 引数
| 名前    | 型  | 説明      |
| ------- | --- | --------- |
| task_id | str | タスク ID |

#### 戻り値
| 型                                                                           | 説明                             |
| ---------------------------------------------------------------------------- | -------------------------------- |
| tuple\[[BulkBody](../04.api/11.schemas.md#bulkbody),Literal\[200]]           | タスクの status                  |
| tuple\[[ErrorResponse](../04.api/11.schemas.md#errorresponse),Literal\[404]] | task_id のタスクが存在しない場合 |
| tuple\[[ErrorResponse](../04.api/11.schemas.md#errorresponse),Literal\[500]] | Redis,DB の接続に失敗した場合    |

#### 処理内容
1. `task_id` を引数に `services.bulks:get_validate_task_result` を呼び出しタスクを取得する。
2. `status` が取得できなければ [`ErrorResponse`](../04.api/11.schemas.md#errorresponse) とともに 404 を戻り値として返す。
3. `status` を取得できれば `status` を [`BulkBody`](../04.api/11.schemas.md#bulkbody) に入れて戻り値として返す。

#### 戻り値
| 型                                                                                  | 説明                             |
| ----------------------------------------------------------------------------------- | -------------------------------- |
| tuple\[[ValidateResults](../03.entities/15.bulk.md#validateresults), Literal\[200]] | データの検証結果                 |
| tuple\[[ErrorResponse](../04.api/11.schemas.md#errorresponse), Literal\[404]]       | task_id のタスクが存在しない場合 |
| tuple\[[ErrorResponse](../04.api/11.schemas.md#errorresponse), Literal\[500]]       | Redis,DB の接続に失敗した場合    |

1. `task_id` を引数に `services.bulks:get_validate_task_result` を呼び出しタスクを取得する。

4. 履歴に対する権限があるか確認する。権限がない場合は [`ErrorResponse`](../04.api/11.schemas.md#errorresponse) とともに 403 を戻り値として返す。
5. 履歴 ID を引数に [`services.bulk:get_validate_result`](../05.services/05.bulk.md#get_validate_result) で検証結果を取得する。
6. [ValidateResults](../03.entities/15.bulk.md#validateresults) を戻り値として返す。

## execute
一括操作タスクの実行をおこなうエンドポイント

#### シグネチャ
```python[bulk.py]
@bp.post("/execute")
@login_required
@roles_required(USER_ROLES.SYSTEM_ADMIN, USER_ROLES.REPOSITORY_ADMIN)
@validate(response_by_alias=True)
@require_enabled("enable_bulk_operation")
def execute(body: ExcuteRequest) -> tuple[BulkBody | ErrorResponse, int]:
```

#### エンドポイント
**POST** /api/bulk/execute/<task_id>

#### 依存するライブラリ
- **Flask**： Blueprint を使用する。
- **Flask-Login**： ログインユーザーの認証を行う。
- **Flask-Pydantic**： リクエストとレスポンスのバリデーション、シリアライズ・デシリアライズを行う。

#### デコレータ
- `@bp.post`{lang=python}： POST リクエストを処理するエンドポイントを定義する。
- `@login_required`{lang=python}： ログインを要求する。
- `@roles_required`{lang=python}： ログインユーザーに「システム管理者」または「リポジトリ管理者」のロールを要求する。
- `@validate`{lang=python}： モデルクラスを使用してリクエストとレスポンスのバリデーションを行う。
- `@require_enabled`{lang=python}： サーバー設定の `enable_bulk_operation` が `True` の場合のみエンドポイントを有効にする。

#### 引数
| 名前 | 型                                             | 説明                             |
| ---- | ---------------------------------------------- | -------------------------------- |
| body | [ExcuteRequest](./11.schemas.md#excuterequest) | 一括操作タスクのリクエストボディ |

#### 戻り値
| 型                                                                           | 説明                 |
| ---------------------------------------------------------------------------- | -------------------- |
| tuple\[[BulkBody](../04.api/11.schemas.md#bulkbody),Literal\[200]]           | タスク ID            |
| tuple\[[ErrorResponse](../04.api/11.schemas.md#errorresponse),Literal\[403]] | 権限がない場合       |
| tuple\[[ErrorResponse](../04.api/11.schemas.md#errorresponse),Literal\[404]] | 履歴が存在しない場合 |

#### 処理内容
1. `temp_file_id` を引数に [`services.history_table.get_history_by_file_id]() を呼び出し履歴 ID を取得する。
2. 権限がない場合は [`ErrorResponse`](../04.api/11.schemas.md#errorresponse) とともに 403 を戻り値として返す。
3. [RecordNotFound](../02.foundation/exc.md#recordnotfound) が発生した場合は [`ErrorResponse`](../04.api/11.schemas.md#errorresponse) とともに 404 を戻り値として返す。
4. [`service.bulk:update_users`](../05.services/05.bulk.md#update_users) を `celery` のタスクで実行する。
5. `task_id` を [BulkBody](../04.api/11.schemas.md#bulkbody) に入れて戻り値として返す。

## result
一括操作情報を取得するエンドポイント

#### シグネチャ
```python[bulk.py]
@bp.get("/result/<string:history_id>")
@login_required
@roles_required(USER_ROLES.SYSTEM_ADMIN, USER_ROLES.REPOSITORY_ADMIN)
@validate(response_by_alias=True)
@require_enabled("enable_bulk_operation")
def result(history_id: UUID, query: UploadQuery) -> tuple[ExecuteResults | ErrorResponse, int]:
```

#### エンドポイント
**GET** /api/bulk/result/<history_id>

#### 依存するライブラリ
- **Flask**： Blueprint を使用する。
- **Flask-Login**： ログインユーザーの認証を行う。
- **Flask-Pydantic**： リクエストとレスポンスのバリデーション、シリアライズ・デシリアライズを行う。

#### デコレータ
- `@bp.get`{lang=python}： GET リクエストを処理するエンドポイントを定義する。
- `@login_required`{lang=python}： ログインを要求する。
- `@roles_required`{lang=python}： ログインユーザーに「システム管理者」または「リポジトリ管理者」のロールを要求する。
- `@validate`{lang=python}： モデルクラスを使用してリクエストとレスポンスのバリデーションを行う。
- `@require_enabled`{lang=python}： サーバー設定の `enable_bulk_operation` が `True` の場合のみエンドポイントを有効にする。

#### 引数
| 名前       | 型                                                   | 説明                  |
| ---------- | ---------------------------------------------------- | --------------------- |
| history_id | uuid.UUID                                            | アップロード履歴の id |
| query      | [`UploadQuery`](../04.api/11.schemas.md#uploadquery) | フィルター項目        |

#### 戻り値
| 型                                                                              | 説明                             |
| ------------------------------------------------------------------------------- | -------------------------------- |
| tuple\[[ExecuteResults](../04.api/11.schemas.md#executeresults), Literal\[200]] | 一括操作結果データ               |
| tuple\[[ErrorResponse](../04.api/11.schemas.md#errorresponse), Literal\[404]]   | 履歴 ID のデータが存在しない場合 |

#### 処理内容
1. query をもとにフィルターを文字列のリストに変換，ページサイズ，オフセットを取り出す
2. 権限がない場合は [`ErrorResponse`](../04.api/11.schemas.md#errorresponse) とともに 403 を戻り値として返す。
3. [`services.bulk:get_upload_result`](../05.services/05.bulk.md#get_upload_result) でデータを取得する。
4. [RecordNotFound](../02.foundation/exc.md#recordnotfound) が発生した場合は [`ErrorResponse`](../04.api/11.schemas.md#errorresponse) とともに 404 を戻り値として返す。
5. データをもとに [ExecuteResults](../03.entities/15.bulk.md#executeresults) を作成し戻り値として返す。
