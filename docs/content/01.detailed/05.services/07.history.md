## get_upload_history_data
表示するアップロード履歴データを取得

#### シグネチャ
```python[history.py]
def get_upload_history_data(criteria: HistoryCriteria) -> SearchResult[UploadHistoryData]:
```

#### 依存するライブラリ
- **SQLAlchemy**： データベースからレコードを取得するために使用。
- **Pydantic**： モデル定義および JSON シリアライズ・デシリアライズに使用。

#### 引数
| 名前     | 型                                  | 説明           |
| -------- | ----------------------------------- | -------------- |
| criteria | [HistoryCriteria](#historycriteria) | フィルター条件 |

#### 戻り値
| 型                                                                                        | 説明       |
| ----------------------------------------------------------------------------------------- | ---------- |
| SearchResult\[[UploadHistoryData](../03.entities/14.history-detail.md#uploadhistorydata)] | 履歴データ |

#### 処理内容
1. `criteria` を引数に [`_build_filters_for_history`](#_build_filters_for_history) を呼び出しフィルターを作成する。
2. [`upload_history`](../../03.db/02.history.md#upload_history-テーブル) テーブルにフィルターを適用し履歴データを取得する。
3. `SQLAlchemyError` が発生した場合は例外をログに記録し、 [DatabaseError](../02.foundation/exc.md#databaseerror) を送出する。
4. [`SearchResult`](../03.entities/07.search-request.md#searchresult) を戻り値として返す。

## _build_filters_for_history
履歴データのクエリに使用するフィルターを構築する

#### シグネチャ
```python[history.py]
def _build_filters_for_history(criteria: HistoryCriteria, history_type: t.Literal["upload", "download"]) -> list[FilterArg]:
```

#### 依存するライブラリ
- **SQLAlchemy**： データベースクエリのフィルターを構築するために使用。

#### 引数
| 名前         | 型                                  | 説明               |
| ------------ | ----------------------------------- | ------------------ |
| criteria     | [HistoryCriteria](#historycriteria) | フィルター条件     |
| history_type | Literal\["upload","download"]       | 履歴データのタイプ |

#### 戻り値
| 型              | 説明                               |
| --------------- | ---------------------------------- |
| list[FilterArg] | クエリに使用するフィルターのリスト |

#### 処理内容
1. システム管理者であれば `criteria.r` を リポジトリ管理者であれば 権限があるリポジトリと `criteria.r` の論理積をフィルターに追加する。
2. その他の `criteria` で指定された値に応じたフィルターを追加する。
3. フィルターを戻り値として返す。

## get_filter_items
フィルターの選択肢を取得する

#### シグネチャ
```python[history.py]
def get_filter_items(tab: t.Literal["download", "upload"], key: str, criteria: OperatorsCriteria) -> SearchResult[UserSummary]:
```

#### 依存するライブラリ
- **SQLAlchemy**： データベースからレコードを取得するために使用。

#### 引数
| 名前     | 型                                      | 説明               |
| -------- | --------------------------------------- | ------------------ |
| tab      | Literal["upload","download"]            | 履歴データのタイプ |
| key      | str                                     | フィルターのキー   |
| criteria | [OperatorsCriteria](#operatorscriteria) | フィルターの条件   |

#### 戻り値
| 型        | 説明                                 |
| --------- | ------------------------------------ |
| list[str] | 実行ユーザーのフィルター選択肢リスト |
| list[str] | 対象グループのフィルター選択肢リスト |
| list[str] | 対象ユーザーのフィルター選択肢リスト |

#### 処理内容
1. `tab=="upload"` ならば [`upload_history`](../../03.db/02.history.md#upload_history-テーブル) テーブル、 そうでなければ [`download_history`](../../03.db/02.history.md#download_history-テーブル) テーブルを対象とする。
2. `key=="O"` ならば `operator_id`、 `operator_name` を対象に `criteria` をもとにフィルターリングしたデータを取得する。
3. `operator_id`、 `operator_name` をもとに [`UserSummary`](../03.entities/10.summaries.md#usersummary) のリストを作成する。
4. `criteria` のをもとに ページサイズやオフセットを含め [`SearchResult`](../03.entities/07.search-request.md#searchresult) を作成し戻り値として返す。


## update_public_status
履歴データのpublicを変更する

#### シグネチャ
```python[history.py]
def update_public_status(*, tab: t.Literal["download", "upload"], history_id: UUID, public: bool) -> bool:
```

#### 依存するライブラリ
- **SQLAlchemy**： データベースのレコードを操作するために使用。

#### 引数
| 名前         | 型                            | 説明                   |
| ------------ | ----------------------------- | ---------------------- |
| id           | uuid.UUID                     | 変更する履歴データのid |
| public       | bool                          | 設定する値             |
| history_type | Literal\["upload","download"] | 履歴データのタイプ     |

#### 戻り値
| 型   | 説明               |
| ---- | ------------------ |
| bool | 変更後のpublicの値 |

#### 処理内容
1. history_typeのテーブルに対してidの履歴データの公開設定をpublicに設定する。
2. `SQLAlchemyError` が発生した場合は例外をログに記録し、 [DatabaseError](../02.foundation/exc.md#databaseerror) を送出する。
3. 変更後の公開設定の値を戻り値として返す。


## get_file_path
ファイルのパスを取得する

#### シグネチャ
```python[history.py]
def get_file_path(file_id: UUID) -> str:
```

#### 依存するライブラリ
- **SQLAlchemy**： データベースからレコードを取得するために使用。

#### 引数
| 名前    | 型        | 説明                   |
| ------- | --------- | ---------------------- |
| file_id | uuid.UUID | 変更する履歴データのid |

#### 戻り値
| 型  | 説明           |
| --- | -------------- |
| str | ファイルのパス |

#### 処理内容
1. `file_id` をもとに [`files`](../../03.db/02.history.md#files-テーブル) テーブルからレコードを取得する。
2. `SQLAlchemyError` が発生した場合は例外をログに記録し、 [DatabaseError](../02.foundation/exc.md#databaseerror) を送出する。
3. レコードが存在しない場合は [`RecordNotFound`](../02.foundation/exc.md#recordnotfound) を送出する。
4. レコードから `file_path` を取得し戻り値として返す。
